Chat POC Application User Guide

Getting Started

The Chat POC Application is a conversational AI system powered by Amazon Bedrock Agent Core. This guide will help you understand how to use the application effectively.

Creating a Chat Session

To start chatting, you need to create a session first:

  1. Send a POST request to /api/sessions
  2. Include your user ID in the request body
  3. Save the returned session ID for future messages

Sending Messages

Once you have a session, connect to the WebSocket endpoint:

  • WebSocket URL: ws://localhost:8080/api/chat
  • Send JSON messages with session_id, content, and timestamp
  • Receive streaming responses in real-time

API Reference

Session Endpoints

POST /api/sessions - Create a new chat session

GET /api/sessions/{id} - Retrieve session information

WebSocket Messages

Send messages in this format:

{
  "session_id": "your-session-id",
  "content": "Your message here",
  "timestamp": "2025-12-11T12:00:00Z"
}

Error Handling

The application provides comprehensive error handling:

  • INVALID_INPUT - Check your request format
  • SESSION_NOT_FOUND - Create a new session
  • RATE_LIMIT_EXCEEDED - Wait before sending more messages
  • SERVICE_ERROR - Temporary service issue, try again

Best Practices

  • Keep messages under 25,000 characters
  • Handle WebSocket reconnections gracefully
  • Implement proper error handling in your client
  • Use session timeouts appropriately (30 minutes)

Troubleshooting

If you encounter issues:

  1. Check your network connection
  2. Verify your session ID is valid
  3. Ensure proper JSON formatting
  4. Check the application logs for detailed error information
